# Git Workflow и Conventional Commits

Проект использует **Conventional Commits** для стандартизации сообщений коммитов.

## Структура коммита

```
<type>: <subject>
```

- **`type`** - тип изменения
- **`subject`** - краткое описание

## Типы коммитов

- **`feat`** - новая функциональность
- **`fix`** - исправление бага
- **`chore`** - рутинные задачи (зависимости, конфигурация)
- **`docs`** - документация
- **`style`** - форматирование кода (не CSS/SCSS)
- **`refactor`** - рефакторинг без изменения функциональности
- **`perf`** - улучшение производительности
- **`build`** - сборка проекта
- **`ci`** - настройка CI/CD
- **`revert`** - откат изменений

## Правила написания

1. Используйте императив (добавить, исправить, обновить)
2. Не начинайте с заглавной буквы, не ставьте точку в конце
3. Ограничьте длину 50-72 символа
4. Пишите на английском языке

**Примеры:**

```bash
feat: add Modal component
fix: fix disabled state styles
chore: update dependencies
```
